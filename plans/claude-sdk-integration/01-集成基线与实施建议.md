# 集成基线与实施建议

## 1. 目标

基于 [`docs/hermes-integration-impact-analysis.md`](/Users/insunny/Documents/codespace/we-claw/docs/hermes-integration-impact-analysis.md) 的推荐顺序，先完成 Claude Agent SDK 的最小可运行 runtime 集成。

本轮目标不是重做前端，而是在现有 Vite + Node launcher 架构中新增中性 runtime bridge，让 UI 可以在 `WE_CLAW_RUNTIME=claude-agent-sdk` 时连接 Claude Agent SDK。

## 2. 边界

首期包含：

- 安装并使用 `@anthropic-ai/claude-agent-sdk`
- 新增 runtime bootstrap 字段和 `/api/runtime/ws`
- 支持 `sessions.list`、`sessions.create`、`chat.history`、`chat.send`、`chat.abort`
- 使用 `query()` + `includePartialMessages` 消费流式输出
- 使用 `listSessions()` / `getSessionMessages()` 读取 Claude 本地 session
- 通过 `AbortController` 和 `Query.interrupt()` 做中断请求

首期不包含：

- inline approval 完整交互
- Claude fork session UI
- Hermes / CLI fallback runtime
- 多 runtime 同屏连接
- 文件 diff、checkpoint rewind、subagent 嵌套视图

## 3. 实施选择

We-Claw 浏览器端继续发内部 Gateway-like frame：

```json
{ "type": "req", "id": "1", "method": "chat.send", "params": {} }
```

Node launcher 在 `/api/runtime/ws` 内分流：

- `openclaw`：复用现有 Gateway bridge
- `claude-agent-sdk`：直接调用 TypeScript SDK
- 其他 runtime：返回未实现诊断

Claude Agent SDK 会话使用 `claude:<sessionId>` 作为 We-Claw session key，避免与 OpenClaw session id 冲突。UI 先创建的空工作使用 `claude:pending:<uuid>`，首次 `query()` 返回真实 `session_id` 后再保存在 launcher 内存映射中。

## 4. 风险

- SDK 认证依赖 `ANTHROPIC_API_KEY` 或受支持云供应商认证；We-Claw 不读取或暴露密钥。
- `permissionMode` 默认使用 `dontAsk`，避免 headless bridge 被权限 prompt 阻塞；需要工具自动执行时通过 `WE_CLAW_CLAUDE_SDK_ALLOWED_TOOLS` 显式配置。
- 首期 session 映射保存在 launcher 内存中；刷新浏览器可通过 SDK 持久 session 恢复历史，但 launcher 重启后 pending session 会丢失。
- `chat.abort` 语义取决于 SDK 当前版本对 `AbortController` / `interrupt()` 的支持。
