# 验收清单

## 1. 自动化

- [ ] `npm run typecheck`
- [ ] `npm test`
- [ ] `npm run build`

## 2. OpenClaw / Claude 回归

- [ ] 默认 `WE_CLAW_RUNTIME` 未设置时仍连接 OpenClaw
- [ ] `/api/gateway/ws` 兼容路径仍可用
- [ ] `/api/runtime/ws` 在 OpenClaw 模式下可复用 Gateway bridge
- [ ] `WE_CLAW_RUNTIME=claude-agent-sdk` 仍可完成 `sessions.list`
- [ ] Claude Agent SDK runtime 仍可完成 `chat.send`、stream delta/final、`chat.history` 和 `chat.abort`

## 3. Hermes 手动验收

- [ ] 使用 `WE_CLAW_RUNTIME=hermes` 启动
- [ ] 使用显式 Hermes 配置启动：`WE_CLAW_HERMES_ROOT`、`WE_CLAW_HERMES_CWD`、`WE_CLAW_HERMES_PYTHON`
- [ ] `/api/bootstrap` 返回 `runtime.kind=hermes`、`transport=stdio-jsonrpc`、`bridgePath=/api/runtime/ws`
- [ ] `sessions.list` 能列出 Hermes 持久 session
- [ ] 创建新工作后能捕获 Hermes active `session_id`
- [ ] 新工作在 UI 中使用 `hermes:<persistedSessionId>` 作为 session key
- [ ] 发送 prompt 后能看到 `message.delta` 或 `message.complete` 输出到 conversation
- [ ] `message.complete` 后 UI running 状态结束
- [ ] 刷新 session list 后能看到 Hermes 持久 session
- [ ] 切换历史 session 时能先 `session.resume`，再加载 `session.history`
- [ ] 历史 session resume 后继续发送 prompt 使用 active Hermes `session_id`
- [ ] 长任务可请求停止，`chat.abort` 能调用 `session.interrupt` 并结束 UI running 状态
- [ ] Hermes stderr / 启动失败 / protocol error 能显示为 diagnostic，而不是静默断连

## 4. 已知缺口

- inline approval 首期未完成；`approval.request` 只做 unsupported / partial 能力提示或轻量 notice。
- clarify / sudo / secret 首期未完成；如果 Hermes run 等待输入，UI 需要明确显示等待状态，避免看起来像卡死。
- Hermes tool activity 首期只要求 compact row 或 notice，不要求完整结构化参数、结果和耗时。
- Hermes Dashboard / FastAPI `/api/ws` 不作为首期验收路径；首期只验收 `python -m tui_gateway.entry` 的 stdio JSON-RPC adapter。
- Hermes 有 persisted session id 与 active runtime session id 两层语义；验收必须覆盖历史 resume 后继续发送和中断，避免只验证新建会话 happy path。
- `WE_CLAW_HERMES_ROOT`、Python virtualenv 和本机依赖会影响真实 E2E；如果本机环境不可用，需要记录启动失败 diagnostic，而不是把自动化测试标记为通过。
