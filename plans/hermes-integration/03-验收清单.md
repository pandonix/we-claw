## 4. 已知缺口

- inline approval 首期未完成；`approval.request` 只做 unsupported / partial 能力提示或轻量 notice。
- clarify / sudo / secret 首期未完成；如果 Hermes run 等待输入，UI 需要明确显示等待状态，避免看起来像卡死。
- Hermes tool activity 首期只要求 compact row 或 notice，不要求完整结构化参数、结果和耗时。
- Hermes history 中纯 tool call、image attach 或 metadata-only message 仍没有完整结构化还原；首期可能显示为 unsupported content 或通过 tool row/notice 降级。
- Hermes Dashboard / FastAPI `/api/ws` 不作为首期验收路径；首期只验收 `python -m tui_gateway.entry` 的 stdio JSON-RPC adapter。
- Hermes stdio client 当前按 Hermes TUI Gateway 的 newline-delimited JSON-RPC 约定实现，不支持 LSP `Content-Length` framing；真实 Hermes lifecycle 验证需要确认该 wire format 没有漂移。
- Hermes 有 persisted session id 与 active runtime session id 两层语义；验收必须覆盖历史 resume 后继续发送和中断，避免只验证新建会话 happy path。
- `WE_CLAW_HERMES_ROOT`、Python virtualenv 和本机依赖会影响真实 E2E；如果本机环境不可用，需要记录启动失败 diagnostic，而不是把自动化测试标记为通过。
