# 集成基线与实施建议

## 1. 目标

基于 [`docs/hermes-integration-impact-analysis.md`](/Users/insunny/Documents/codespace/we-claw/docs/hermes-integration-impact-analysis.md) 的 Hermes 设计判断，完成 Hermes runtime 的最小可运行集成计划。

本轮目标不是嵌入 Hermes Dashboard，也不是重写 Hermes TUI，而是在现有 Vite + Node launcher 架构中新增 Hermes adapter，让 UI 可以在 `WE_CLAW_RUNTIME=hermes` 时通过 We-Claw 内部 runtime bridge 连接 Hermes TUI Gateway。

首期应把 Hermes 当作一个显式启用的本地 runtime。OpenClaw 仍保持默认 runtime，Claude Agent SDK 已有专项计划和实现路径，本计划只处理 Hermes 通过 `stdio-jsonrpc` 接入 We-Claw 工作台所需的边界、映射和验收重点。

## 2. 边界

首期包含：

- 新增 Hermes runtime 配置：`WE_CLAW_RUNTIME=hermes`、`WE_CLAW_HERMES_PYTHON`、`WE_CLAW_HERMES_ROOT`、`WE_CLAW_HERMES_CWD`、`WE_CLAW_HERMES_STARTUP_TIMEOUT_MS`
- Node launcher 管理 Hermes Python 子进程，启动 `python -m tui_gateway.entry`
- 为 Hermes 设置 `PYTHONPATH`、工作目录和启动超时，并把 stderr 收敛为 launcher diagnostics
- 在现有 `/api/runtime/ws` 内新增 Hermes 分支，浏览器仍使用 We-Claw 内部 `{ type: "req", method, params }` frame
- 支持 `sessions.list`、`sessions.create`、`chat.history`、`chat.send`、`chat.abort`
- 在 adapter 内完成 We-Claw 方法到 Hermes JSON-RPC 方法的翻译：`session.list`、`session.create`、`session.resume`、`session.history`、`prompt.submit`、`session.interrupt`
- 归一化 Hermes 基础事件：`message.start`、`message.delta`、`message.complete`、`error`、`status.update`
- 维护 Hermes 持久 session id 与活跃 runtime session id 的映射，保证历史会话 resume 后可以继续发送 prompt

首期不包含：

- 嵌入 Hermes Dashboard / FastAPI `/api/ws`
- 复制 Hermes TUI 的整套聊天 UI、slash command UI 或配置页
- Hermes tool activity 的完整结构化展示
- `approval.request`、`clarify.request`、`sudo.request`、`secret.request` 的 inline 交互闭环
- Hermes plugins、cron、skills、keys、logs 管理界面
- 多 runtime 同屏连接或跨 runtime session 合并
- Claude Agent SDK / CLI fallback 的新增工作

## 3. 实施选择

We-Claw 浏览器端继续发内部 Gateway-like frame：

```json
{ "type": "req", "id": "1", "method": "chat.send", "params": {} }
```

Node launcher 在 `/api/runtime/ws` 内分流：

- `openclaw`：复用现有 Gateway bridge
- `claude-agent-sdk`：继续走已存在的 TypeScript SDK adapter
- `hermes`：启动并管理 Hermes `tui_gateway.entry` 子进程，通过 JSON-RPC over stdio 与 Python runtime 通讯
- 其他 runtime：返回未实现诊断

Hermes 不应直接暴露给浏览器。Node 侧 adapter 负责把 We-Claw request 翻译成 Hermes JSON-RPC：

| We-Claw 语义 | Hermes 方法 | 首期行为 |
| --- | --- | --- |
| `sessions.list` | `session.list` | 返回可展示的持久会话列表 |
| `sessions.create` | `session.create` | 创建活跃 Hermes session，并记录映射 |
| `chat.history` | `session.resume` + `session.history` | 必要时先 resume，再读取历史 |
| `chat.send` | `prompt.submit` | 向活跃 session 发送 prompt，并消费事件流 |
| `chat.abort` | `session.interrupt` | 请求中断当前活跃 session |

Hermes 会话使用 `hermes:<persistedSessionId>` 作为 We-Claw session key，避免与 OpenClaw / Claude session id 冲突。adapter 内部维护：

```ts
persistedSessionId -> activeHermesSessionId
```

当用户打开历史会话但没有 active mapping 时，adapter 应先调用 `session.resume`，再把返回的活跃 `session_id` 用于后续 `prompt.submit`、`session.history` 和 `session.interrupt`。UI rail 始终展示持久 session key，避免刷新后绑定到只在进程内有效的活跃 id。

Hermes event 首期只映射基础聊天生命周期：

- `message.start`：创建或标记 running assistant message
- `message.delta`：追加 assistant 文本
- `message.complete`：收敛 assistant final 状态
- `status.update`：更新 runtime status 或生成轻量 notice
- `error`：生成 error message 并结束当前 running 状态

`tool.start`、`tool.progress`、`tool.complete` 可以先降级为 compact tool row 或 runtime notice；approval / clarify / sudo / secret 请求首期不做交互闭环，但必须在能力模型里标记为 unsupported 或 partial，避免 UI 展示可用但实际会阻塞的控制。

## 4. 风险

- Hermes `tui_gateway` 是 TUI 内部协议，不应假设它与 OpenClaw Gateway 一样稳定；adapter 需要把方法名、事件名和 payload parsing 集中在单独模块。
- Hermes 有持久 session id 与活跃 runtime session id 两层语义；如果映射处理不完整，会出现历史能列出但不能继续发送、或 abort 指向错误 session 的问题。
- Hermes 子进程生命周期由 Node launcher 管理；需要明确启动失败、stderr 输出、进程退出、浏览器断连和 We-Claw 关闭时的清理路径。
- `prompt.submit` 与事件流的时序需要真实验证，不能假设所有 assistant 文本都会以 `message.delta` 到达；`message.complete` 也可能携带最终内容或状态。
- approval / clarify / sudo / secret 请求如果被忽略，Hermes run 可能停在等待输入状态；首期应通过 capabilities 降级和 status 提示显式表达。
- `WE_CLAW_HERMES_ROOT`、`PYTHONPATH`、Python virtualenv 与本机依赖版本会影响可用性；bootstrap diagnostic 需要区分"Hermes 未配置"、"Python 启动失败"和"Gateway 方法不可用"。
- Hermes Dashboard 路径虽然存在 WebSocket handler，但涉及 token、embedded chat flag 和 Dashboard 生命周期；首期坚持 stdio adapter，避免把 We-Claw 变成 Hermes Dashboard 外壳。
- OpenClaw 是默认 runtime；Hermes 接入必须验证默认启动、`/api/gateway/ws` 兼容路径和 Claude SDK runtime 都不回退。
